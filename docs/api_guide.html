DataSchema Base Class¶

class DataSchemaConcrete(only=None, exclude=(), many=False, normalize_many=False, context=None, load_only=(), dump_only=(), partial=False, unknown=None, load_dataclass=True, use_defaults=False, fast_dumps=False)[source]¶

Generically-typed subclass of marshmallow.Schema.

Allows for Generic typing with model for help with autom-completion and mypy

>>> schema: DataSchema[Book] = dataclass_schema(Book)

or:

>>> @schema_for(Book)
... class BookSchema(DataSchema[Book])
...     pass

DataSchema implements sane default for serializing / deserializing dataclasses using marshmallow’s post_load and pre-dump methods. See below.

.load(), .loads(), .dump(), .dumps() and .validate() are re-implemented with no alterations to add type-hinting for a more robust mypy experience.

UPDATE_ALLOWED: List[str] = []¶

Some classes may have a very limited number of attributes that are allowed to be updated in a PATCH. This attribute can be used as a consistent way to notate them. To be passed to init as only param when checking incoming objects in these methods. It has no actual effect on the schema by itself.

WRITE_PROTECTED: List[str] = []¶

This attribute is reserved for fields that should be protected in write-methods like a POST or PATCH. To be passed to init as exclude param when checking incoming objects for these methods. It has no actual effect on the schema by itself.

Garams dataclass¶

class Garams[source]¶

All attributed correspond to a param of marshmallow.fields.Field. All fields have a default value of DEFAULT, and will not overwrite the auto-calculated values from grahamcracker.

Passing None to a value will force the param to None when it is passed to the field.

The field names and their possible values are listed here, for more information on how they affect a field, see the marshmallow documentation.

Fields:
